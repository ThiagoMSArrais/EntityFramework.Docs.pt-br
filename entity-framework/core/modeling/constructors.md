## <a name="binding-to-mapped-properties"></a><span data-ttu-id="5fa19-108">Associando a propriedades mapeadas</span><span class="sxs-lookup"><span data-stu-id="5fa19-108">Binding to mapped properties</span></span>

<span data-ttu-id="5fa19-109">Considere um modelo típico de postagem do Blog:</span><span class="sxs-lookup"><span data-stu-id="5fa19-109">Consider a typical Blog/Post model:</span></span>

```Csharp
public class Blog
{
    public int Id { get; set; }

    public string Name { get; set; }
    public string Author { get; set; }

    public ICollection<Post> Posts { get; } = new List<Post>();
}

public class Post
{
    public int Id { get; set; }
    
    public string Title { get; set; }
    public string Content { get; set; }
    public DateTime PostedOn { get; set; }

    public Blog Blog { get; set; }
}
```

<span data-ttu-id="5fa19-110">Quando o EF Core cria instâncias desses tipos, como para os resultados de uma consulta, ele primeiro chamar o construtor sem parâmetros padrão e, em seguida, defina cada propriedade para o valor do banco de dados.</span><span class="sxs-lookup"><span data-stu-id="5fa19-110">When EF Core creates instances of these types, such as for the results of a query, it will first call the default parameterless constructor and then set each property to the value from the database.</span></span> <span data-ttu-id="5fa19-111">No entanto, se EF Core localiza um construtor com parâmetros com nomes de parâmetro e tipos que correspondem às de mapeados propriedades, ele chamará em vez disso, o construtor com parâmetros com valores para as propriedades e não definirá explicitamente cada propriedade.</span><span class="sxs-lookup"><span data-stu-id="5fa19-111">However, if EF Core finds a parameterized constructor with parameter names and types that match those of mapped properties, then it will instead call the parameterized constructor with values for those properties and will not set each property explicitly.</span></span> <span data-ttu-id="5fa19-112">Por exemplo:</span><span class="sxs-lookup"><span data-stu-id="5fa19-112">For example:</span></span>

```Csharp

{
    public Blog(int id, string name, string author)
    {
        Id = id;
        Name = name;
        Author = author;
    }

    public int Id { get; set; }

    public string Name { get; set; }
    public string Author { get; set; }

    public ICollection<Post> Posts { get; } = new List<Post>();
}

public class Post
{
    public Post(int id, string title, DateTime postedOn)
    {
        Id = id;
        Title = title;
        PostedOn = postedOn;
    }

    public int Id { get; set; }

    public string Title { get; set; }
    public string Content { get; set; }
    public DateTime PostedOn { get; set; }

    public Blog Blog { get; set; }
}
```
<span data-ttu-id="5fa19-113">Algumas coisas a observar:</span><span class="sxs-lookup"><span data-stu-id="5fa19-113">Some things to note:</span></span>
* <span data-ttu-id="5fa19-114">Nem todas as propriedades precisam ter parâmetros do construtor.</span><span class="sxs-lookup"><span data-stu-id="5fa19-114">Not all properties need to have constructor parameters.</span></span> <span data-ttu-id="5fa19-115">Por exemplo, a propriedade Post.Content não é definida por qualquer parâmetro de construtor, para que Core EF definirá depois de chamar o construtor da maneira normal.</span><span class="sxs-lookup"><span data-stu-id="5fa19-115">For example, the Post.Content property is not set by any constructor parameter, so EF Core will set it after calling the constructor in the normal way.</span></span>
* <span data-ttu-id="5fa19-116">Os nomes e tipos de parâmetro devem corresponder tipos de propriedade e nomes, exceto que as propriedades podem ser maiusculas e minúsculas de Pascal enquanto os parâmetros são concatenados.</span><span class="sxs-lookup"><span data-stu-id="5fa19-116">The parameter types and names must match property types and names, except that properties can be Pascal-cased while the parameters are camel-cased.</span></span>
* <span data-ttu-id="5fa19-117">EF Core não é possível definir propriedades de navegação (como Blog ou postagens acima) usando um construtor.</span><span class="sxs-lookup"><span data-stu-id="5fa19-117">EF Core cannot set navigation properties (such as Blog or Posts above) using a constructor.</span></span>
* <span data-ttu-id="5fa19-118">O construtor pode ser público, privada, ou qualquer outro tipo de acessibilidade.</span><span class="sxs-lookup"><span data-stu-id="5fa19-118">The constructor can be public, private, or have any other accessibility.</span></span>

### <a name="read-only-properties"></a><span data-ttu-id="5fa19-119">Propriedades somente leitura</span><span class="sxs-lookup"><span data-stu-id="5fa19-119">Read-only properties</span></span>

<span data-ttu-id="5fa19-120">Depois que as propriedades estão sendo definidas via construtor pode fazer sentido fazer algumas delas somente leitura.</span><span class="sxs-lookup"><span data-stu-id="5fa19-120">Once properties are being set via the constructor it can make sense to make some of them read-only.</span></span> <span data-ttu-id="5fa19-121">EF Core dá suporte a isso, mas há algumas coisas para verificar:</span><span class="sxs-lookup"><span data-stu-id="5fa19-121">EF Core supports this, but there are some things to look out for:</span></span>
* <span data-ttu-id="5fa19-122">Propriedades sem setters não mapeadas pela convenção.</span><span class="sxs-lookup"><span data-stu-id="5fa19-122">Properties without setters are not mapped by convention.</span></span> <span data-ttu-id="5fa19-123">(Isso tende mapear propriedades que não devem ser mapeadas, como as propriedades calculadas).</span><span class="sxs-lookup"><span data-stu-id="5fa19-123">(Doing so tends to map properties that should not be mapped, such as computed properties.)</span></span>
* <span data-ttu-id="5fa19-124">Usar valores de chave gerados automaticamente requer uma propriedade de chave é leitura / gravação, pois o valor da chave precisa ser definida pelo gerador de chave ao inserir novas entidades.</span><span class="sxs-lookup"><span data-stu-id="5fa19-124">Using automatically generated key values requires a key property that is read-write, since the key value needs to be set by the key generator when inserting new entities.</span></span>

<span data-ttu-id="5fa19-125">Uma maneira fácil de evitar essas coisas é usar setters privadas.</span><span class="sxs-lookup"><span data-stu-id="5fa19-125">An easy way to avoid these things is to use private setters.</span></span> <span data-ttu-id="5fa19-126">Por exemplo:</span><span class="sxs-lookup"><span data-stu-id="5fa19-126">For example:</span></span>

<span data-ttu-id="5fa19-127">Núcleo EF vê uma propriedade com um setter privada como leitura-gravação, o que significa que todas as propriedades são mapeadas como antes e a chave pode ainda ser gerado pelo repositório.</span><span class="sxs-lookup"><span data-stu-id="5fa19-127">EF Core sees a property with a private setter as read-write, which means that all properties are mapped as before and the key can still be store-generated.</span></span>

<span data-ttu-id="5fa19-132">Observe o seguinte:</span><span class="sxs-lookup"><span data-stu-id="5fa19-132">Things to note:</span></span>

## <a name="injecting-services"></a><span data-ttu-id="5fa19-140">Injeção de serviços</span><span class="sxs-lookup"><span data-stu-id="5fa19-140">Injecting services</span></span>

<span data-ttu-id="5fa19-149">Por exemplo, um DbContext injetado pode ser usado para acessar seletivamente o banco de dados para obter informações sobre entidades relacionadas sem carregar todos eles.</span><span class="sxs-lookup"><span data-stu-id="5fa19-149">For example, an injected DbContext can be used to selectively access the database to obtain information about related entities without loading them all.</span></span> <span data-ttu-id="5fa19-150">No exemplo a seguir, isso é usado para obter o número de mensagens em um blog sem carregar as postagens:</span><span class="sxs-lookup"><span data-stu-id="5fa19-150">In the example below this is used to obtain the number of posts in a blog without loading the posts:</span></span>

```Csharp
public class Blog
{
    public Blog()
    {
    }

    private Blog(BloggingContext context)
    {
        Context = context;
    }

    private BloggingContext Context { get; set; }

    public int Id { get; set; }
    public string Name { get; set; }
    public string Author { get; set; }

    public ICollection<Post> Posts { get; set; }

    public int PostsCount
        => Posts?.Count
           ?? Context?.Set<Post>().Count(p => Id == EF.Property<int?>(p, "BlogId"))
           ?? 0;
}

public class Post
{
    public int Id { get; set; }
    public string Title { get; set; }
    public string Content { get; set; }
    public DateTime PostedOn { get; set; }

    public Blog Blog { get; set; }
}
```
<span data-ttu-id="5fa19-151">Algumas coisas a observar sobre isso:</span><span class="sxs-lookup"><span data-stu-id="5fa19-151">A few things to notice about this:</span></span>
